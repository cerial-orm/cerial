---
title: findAll
description: Convenience method to fetch all records from a table with no filtering options.
---

Returns all records from a table. This is a convenience alias for [`findMany()`](/queries/find-many) with no options.

```typescript
const users = await client.db.User.findAll();
// users: User[]
```

## Usage

`findAll()` accepts no arguments — no `where`, `select`, `include`, `orderBy`, `limit`, or `offset`.

```typescript
const allUsers = await client.db.User.findAll();

for (const user of allUsers) {
  console.log(user.name);
}
```

## Return Type

`findAll()` always returns the full model type array:

| Scenario | Return |
| --- | --- |
| Records exist | `T[]` (array of full model objects) |
| Table is empty | `[]` (empty array) |

## Equivalence

`findAll()` is exactly equivalent to calling `findMany()` with no arguments:

```typescript
// These are identical
const all1 = await client.db.User.findAll();
const all2 = await client.db.User.findMany();
```

## When to Use

Use `findAll()` when you want every record without any filtering. Use [`findMany()`](/queries/find-many) when you need `where`, `select`, `orderBy`, `limit`, `offset`, or `include`.

```typescript
// Use findAll — simple, readable, no options needed
const allUsers = await client.db.User.findAll();

// Use findMany — when you need filtering or pagination
const activeUsers = await client.db.User.findMany({
  where: { isActive: true },
  orderBy: { createdAt: 'desc' },
  limit: 50,
});
```

:::tip
`findAll()` is best for small tables or admin operations. For large tables, prefer `findMany()` with `limit` and `offset` to avoid loading thousands of records into memory.
:::
